# Prototype Instructions

Run the local server yourself and open the preview in the in-app browser. Do not give the user server-start instructions when you can run it.

Before making substantial visual changes, use the Product Design plugin's `get-context` skill when the visual source is unclear or no longer matches the current goal. When the user gives durable prototype-specific design feedback, preferences, or decisions, record them in `AGENTS.md`.

When implementing from a selected generated mock, treat that image as the source of truth for layout, component anatomy, density, spacing, color, typography, visible content, and hierarchy.

Selected direction for this prototype: warm, iOS-like, teacher-note oriented, with uploaded artwork preview, a teacher guidance card, composition skeleton, and next-step drawing suggestions.

Latest design feedback: keep the teacher guidance area simpler. Do not show four separate principle chips/cards in the main note area; keep the focus on the artwork, warm teacher note, observation tags, composition skeleton, and next-step suggestions.

New product direction: in addition to composition guidance, the app should support coloring guidance. Use the user's provided before/after artwork as the product reference: uploaded line draft first, then guided color direction with palette, coloring focus, step order, and an after-color result comparison slot.

Composition guidance should support an AI-generated suggestion image, not only abstract overlay lines. Use the user's animal skating artwork reference as the pattern: child finished artwork first, then an AI composition/skeleton suggestion image showing added scene structure, track lines, background, audience, and environmental context.
